# 🚀 PyThermoLinkDB v1.6.0

This release expands the high-level `mk*` helper API for building thermodynamic data, equations, constants, and multi-component source bundles directly from a `ModelSource`.

## ✨ New Features

### 🧮 Equation Source Builder

Added support for building component equation collections with `mkeqs()`.

- ✅ Build all equations for a component with `build_all=True`
- 🎯 Restrict generated equations with `build_list`, such as `["Cp_IG", "Cp_LIQ"]`
- 🔎 Inspect available equations with `all_available_equations()`, `check_availability()`, and `all_available()`
- 🧩 Select individual equation sources with `select(name=...)`
- 🌡️ Validate equation inputs and units before calculation
- 🔁 Convert runtime inputs using external unit conversion functions such as `pycuc.convert_from_to`

Example: `examples/mks/mk-exp-2.py`

### 📊 Data Source Builder

Added direct single-component data source creation with `mkdt()`.

- 📦 Build a `DataSourceCore` from a component and `ModelSource`
- 🏷️ Resolve components with configurable keys such as `Name-State`
- 📚 List available properties with `props`, `props_symbols`, and `all_props()`
- 🔍 Check property availability with `is_prop_available()`, `check_availability()`, and `all_available()`
- 🎯 Select property data with `prop()` and `select()`

Example: `examples/mks/mkdt - 1.py`

### 🧱 Constants Source Builder

Added constants source creation with `mkct()`.

- 🧾 Build a `ConstantsSourceCore` directly from a `ModelSource`
- 🔢 Access all constants and constant symbols
- ✅ Check constants with `is_constant_available()`, `check_constants_availability()`, and `check_availability()`
- 🎯 Select constants using `constant()`, `const()`, `symbol()`, `const_symbol()`, `prop()`, `select_scalar()`, and `select()`
- 🧩 Support scalar constants, property-like constants, nested dictionaries, lists, and other raw constant values

Example: `examples/mks/mkct - 1.py`

### 🧪 Multi-Component Data Sources

Added multi-component data source creation with `mkdts()`.

- 🧬 Build data sources for multiple components in one call
- 🗂️ Return a dictionary keyed by component IDs generated from the selected `component_key`
- 🔎 Access each component's `DataSourceCore` independently
- 📊 Use the same property discovery, availability, and selection helpers as `mkdt()`

Example: `examples/mks/mkdts - 1.py`

### ⚙️ Multi-Component Equation Sources

Added multi-component equation source creation with `mkeqss()`.

- 🧬 Build equation collections for multiple components in one call
- 🗂️ Return a dictionary keyed by component IDs generated from the selected `component_key`
- 🎯 Build all equations or restrict to a selected `build_list`
- 🔎 Inspect, validate, and select equations per component
- 🌡️ Validate runtime inputs and units for each equation collection
- 🧮 Calculate selected equations after validated input building and unit conversion

Example: `examples/mks/mkeqss - 1.py`

## 🧰 Developer Experience

- 🧭 Examples cover direct source creation, source inspection, availability checks, selection helpers, input validation, and calculation.
