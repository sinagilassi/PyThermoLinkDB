# 🚀 v1.5.0 Release Note: Constants Sources, Unified ModelSource, and Equation Inputs

## ✨ New Features

### 🧱 Constants ThermoDB and Constants Model Source

This release adds a cleaner workflow for building constants into a model source.

- Use `build_constants_thermodb_from_reference` from `pyThermoDB` to extract constants from reference content.
- Use `build_constants_model_source` from `pyThermoLinkDB` to convert a `ConstantsThermoDB` object into a `ConstantsModelSource`.
- Build constants sources with either reference rules or custom rules.

```python
from pyThermoDB import build_constants_thermodb_from_reference
from pyThermoLinkDB import build_constants_model_source

constants_thermodb = build_constants_thermodb_from_reference(
    reference_content=REFERENCE_CONTENT,
)

constants_model_source = build_constants_model_source(
    constants_thermodb=constants_thermodb,
    rules=None,
)
```

🔧 Custom constants rules are also supported:

```python
thermodb_rules = {
    "ALL": {
        "CONSTANTS": {
            "Universal Gas Constant": "R",
            "total heat capacity of ideal gas": "Cp_IG",
            "enthalpy of reaction": "dH_rxn",
            "binary parameter": "Xb",
            "custom constants": "X",
        }
    }
}

constants_model_source = build_constants_model_source(
    constants_thermodb=constants_thermodb,
    rules=thermodb_rules,
)
```

### 🧩 Unified `build_model_source`

`build_model_source` now combines component, mixture, and constants model sources into one `ModelSource`.

📦 The resulting `ModelSource` includes:

- `data_source`
- `equation_source`
- `constants_source`
- `data_symbols`
- `equation_symbols`
- `constants_symbols`

```python
from pyThermoLinkDB import build_model_source

sources = [constants_model_source] + component_model_source

model_source = build_model_source(
    source=sources,
)

datasource = model_source.data_source
equationsource = model_source.equation_source
constantssource = model_source.constants_source
```

The `ModelSource` schema is defined in `pyThermoLinkDB.models.source` and now represents all supported source types in a single object.

### 🛠️ `Source` Runtime Access Layer

The `Source` class in `pyThermoLinkDB.thermo.source` provides a direct access layer for the different sections of a `ModelSource`.

📚 `ModelSource` is now organized into separate source sections:

- `data_source`: component property data.
- `equation_source`: component equation objects.
- `constants_source`: shared constants.
- `data_symbols`: metadata for data source symbols.
- `equation_symbols`: metadata for equation arguments and return symbols.
- `constants_symbols`: metadata for constants.

🔎 Use `Source` when you want to inspect, extract, build, or evaluate model source content:

```python
from pyThermoLinkDB.thermo import Source

source = Source(
    model_source=model_source,
    component_key="Formula-State",
)

data_symbols = source.data_symbols
equation_symbols = source.equation_symbols
constants_symbols = source.constants_symbols
```

⚙️ New source helper methods include:

- `eq_builder`: builds executable equation sources for selected components and properties.
- `eq_eval`, `eval_eq`, and `exec_eq`: evaluate built equation sources with runtime argument values.
- `component_eq_symbols` and `eq_symbol`: inspect equation symbol metadata.
- `get_dt` and `get_dt_symbols`: access all data and data symbols for a component.
- `get_prop` and `get_prop_symbol`: access a specific component property and its symbol metadata.
- `const` and `const_symbol`: access constants and their symbol metadata.
- `is_constant_available`, `is_prop_available`, `is_prop_eq_available`, and `is_prop_data_available`: check source availability.

```python
eq_src = source.eq_builder(
    components=[CO2],
    prop_name="Cp_IG",
    component_keys=["Name-State", "Formula-State", "Name-Formula-State"],
)

eq_res = source.eq_eval(
    components=[CO2],
    eq_src_comp=eq_src,
    args_values={"T": 298.15},
)

comp_data = source.get_dt("CO2-g")
comp_data_symbols = source.get_dt_symbols("CO2-g")

enfo = source.get_prop("CO2-g", "EnFo_IG")
enfo_symbol = source.get_prop_symbol("CO2-g", "EnFo_IG")

has_r = source.is_constant_available("R")
gas_constant = source.const("R")
gas_constant_symbol = source.const_symbol("R")
```

### 🧮 Equation Input Builder Utilities

v1.5.0 introduces utilities for preparing runtime inputs before equation evaluation.

🧰 New helpers in `pyThermoLinkDB.utils.input_builder`:

- `check_unit_availability`: verifies that required equation units are supported by a unit checker.
- `check_inputs_availability`: verifies that required runtime inputs are present.
- `build_inputs`: builds the final input dictionary and converts runtime values to the units expected by the equation.

🔁 Example with `pycuc`:

```python
import pycuc
from pyThermoLinkDB.utils.input_builder import (
    check_inputs_availability,
    check_unit_availability,
    build_inputs,
)

eq_inputs = {
    "T": {"value": 0, "unit": "K"},
    "P": {"value": 0, "unit": "bar"},
    "V": {"value": 0, "unit": "L"},
    "Q": {"value": 0, "unit": "kJ"},
}

inputs = {
    "T": {"value": 25.0, "unit": "C"},
    "P": {"value": 101325.0, "unit": "Pa"},
    "V": {"value": 0.001, "unit": "m3"},
}

units_available, unit_details = check_unit_availability(
    eq_inputs,
    pycuc.is_unit_available,
)

inputs_available, input_details = check_inputs_availability(
    eq_inputs,
    inputs,
)

built_inputs = build_inputs(
    eq_inputs,
    inputs,
    pycuc.convert_from_to,
)
```

`build_inputs` keeps default values for missing optional inputs and converts provided runtime values into the units declared by `eq_inputs`.
